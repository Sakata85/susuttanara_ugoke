# 食事・運動記録アプリ：すすったなら動け！

## このアプリについて

ログインしたユーザーが、日々の食事と運動を記録し、摂取カロリーと消費カロリーの差分を確認できるWebアプリです。
デザインは、作成者の好物である「ラーメン二郎」をリスペクトしたものになっています。

##制作背景

ラーメン二郎が好物ではあるものの、自分の健康も心配ではあるというジレンマから考案しました。

## 技術選定理由

- **Next.js**
  - React ベースのフルスタックフレームワークで、SSR・ルーティング・API が一通り揃い、開発効率がよいため。App Router により認証状態に応じたレイアウトやデータ取得を組み立てやすい。

- **Supabase**
  - 認証・DB（PostgreSQL）・ストレージ・Edge Functions を一括で利用できる BaaS のため。個人開発や小規模サービスでインフラを簡素にしつつ、スキーマや RLS でデータの整合性を保てる。

- **Tailwind CSS**
  - ユーティリティファーストでコンポーネントの見た目を素早く揃えられ、デザインの一貫性を保ちやすいため。rem ベースの調整やレスポンシブも扱いやすい。

- **TypeScript**
  - 型による補完とコンパイル時のチェックでバグを減らし、リファクタや長期運用をしやすくするため。

## できること

- **記録の登録**
  - 食品名・摂取カロリー・体重を入力して記録
  - 運動種別（ウォーキング、ジョギング、縄跳び、水泳など）と実施時間を入力すると、消費カロリーを自動計算
  - 記録に画像を添付可能
  - メモを残せる

- **記録の確認**
  - 自分が登録した記録の一覧を閲覧
  - 各記録の摂取・消費・差分カロリーを表示

- **認証**
  - 会員登録・ログイン・ログアウト
  - パスワード忘れ時の再設定

## 今後の展望

- **履歴のページネーション**
  - 記録が増えたときに一覧をページ分割して表示し、読み込み負荷を抑えつつ管理しやすくする

- **ユーザー定義の運動**
  - ウォーキング・ジョギングなど既定の種別に加え、ユーザーが運動名と消費カロリー（またはMETs）を登録して使えるようにする

- **記録の編集・削除**
  - 登録済みの食事・運動記録を修正したり削除したりできるようにする

- **推移の可視化**
  - 日別・週別・月別の摂取カロリー・消費カロリー・体重などをグラフで表示し、傾向を把握しやすくする

- **目標の設定と進捗**
  - 目標体重や1日の目標カロリーを設定し、達成状況をダッシュボードなどで表示する

- **よく使う項目の登録**
  - 頻繁に使う食品や運動を「お気に入り」やテンプレートとして登録し、記録入力を簡略化する

- **データのエクスポート**
  - 記録をCSVなどで出力し、他のツールでの分析やバックアップに利用できるようにする
